## 1. pyproject.toml

**改了什么**: `torch==2.2.0` → `torch>=2.8.0`, `torchvision==0.17.0` → `torchvision>=0.23.0`, `torchaudio==2.2.0` → `torchaudio>=2.8.0`

**为什么改**: RTX 5090 D 是 Blackwell 架构 (sm_120)，PyTorch 2.2.0/2.6.0/2.7.0 均不包含 sm_120 内核，必须升级到 2.8.0+cu128 才能使用 GPU。同时卸载了 flash-attn（也不支持 sm_120），改用 PyTorch 原生 SDPA (`attn_implementation="sdpa"`)。

## 4. vla-scripts/preprocess_fuxi.py（新建）

**功能**: 将伏羲挖掘机 H5 数据集转换为 OpenVLA 训练用的 TFRecord 格式

**为什么新建**: 伏羲数据是 HDF5 格式，需要转换成 OpenVLA 兼容的数据格式。

**处理步骤**:
1. 读取 30 个 H5 文件（3台挖掘机，各10条）
2. BGR → RGB 转换
3. 缩放至 224×224
4. 计算动作增量 (action[t] - qpos[t])
5. 计算 Q01/Q99 归一化统计量
6. 写入 TFRecord

**输出**: `FuXiData/processed/`（11GB, 75,355 帧）

## 8. prismatic/vla/datasets/excavator_dataset.py（新建）

**功能**: 自定义 PyTorch Dataset，从 TFRecord 读取挖掘机数据

**为什么新建**: 预处理生成的 TFRecord 格式与 OpenVLA 原生 RLDS 管道不兼容，需要自定义加载器。

**实现**: 继承 `torch.utils.data.Dataset`，按 `DummyDataset` 接口返回 `{pixel_values, input_ids, labels}`

## 微调参数推荐（32GB VRAM）

```bash
torchrun --standalone --nnodes 1 --nproc-per-node 1 vla-scripts/finetune.py \
  --vla_path "/home/wika/data/openvla/openvla-7b" \
  --custom_dataset_path "FuXiData/processed/excavator_motion.tfrecord" \
  --run_root_dir "runs" \
  --adapter_tmp_dir "adapter-tmp" \
  --lora_rank 32 \
  --batch_size 2 \
  --grad_accumulation_steps 16 \
  --learning_rate 5e-4 \
  --image_aug False \
  --save_steps 1000
```

| 参数 | 值 | 说明 |
|---|---|---|
| `--lora_rank` | 32 | LoRA 秩，平衡效果与显存 |
| `--batch_size` | 2 | 32GB 显存安全值 (~18GB 实际占用) |
| `--grad_accumulation_steps` | 16 | 等效 batch=32，保梯度稳定 |
| `--learning_rate` | 5e-4 | LoRA 微调标准学习率 |
| `--image_aug` | False | 关闭数据增强，挖掘机数据本身已有重复动作多样性 |
| `--save_steps` | 1000 | 每 1000 个梯度步保存检查点 |

## 11. vla-scripts/finetune.py 后续修复（2026-07-20 16:19）

**改了什么**:
1. 新增 `--no_wandb` 参数（布尔型），禁用 W&B 日志
2. 当 `custom_dataset_path` 设时，`dataset_label` 改为 `"excavator_motion"`
